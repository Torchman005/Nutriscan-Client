## 1. 总体功能描述

NutriScan食品营养AI助手是一款基于Android平台的智能健康管理应用，旨在通过人工智能技术帮助用户管理饮食健康。

**核心功能包括：**
*   **AI智能食物识别**：利用图像识别技术，通过拍照或上传图片，自动识别食物名称、分析主要成分、检测潜在过敏原，并提供专业的食用建议和风险评估。
*   **健康交流社区（论坛）**：提供一个专注于健康饮食的社交平台，用户可以根据养生、健身、幼儿辅食等兴趣群体浏览内容，发布包含图文的健康笔记，并进行点赞、评论和收藏等互动。
*   **个性化健康管理**：建立用户健康档案，记录身高、体重、BMI等数据，生成体重变化周报，并根据用户所属群体（如健身人群、孕妇、幼儿）推荐相关内容。

## 2. 运行环境

### 2.1 硬件环境
*   **终端设备**：Android 智能手机
*   **处理器**：建议骁龙660及以上或同级别处理器
*   **内存**：建议 4GB RAM 及以上
*   **存储空间**：建议预留 200MB 以上可用空间
*   **摄像头**：后置摄像头（用于食物拍照识别）

### 2.2 软件环境
*   **操作系统**：Android 8.0 (Oreo, API Level 26) 及以上版本
*   **网络环境**：需保持互联网连接（Wi-Fi 或 4G/5G），用于与云端服务器通信及加载图片资源。

## 4. 软件安装说明

1.  **获取安装包**：从开发团队或应用分发平台获取 `NutriScan.apk` 安装文件。
2.  **传输文件**：将 APK 文件传输至 Android 手机存储中（如通过 USB、微信文件传输助手等）。
3.  **允许安装未知来源**：
    *   在手机“设置” -> “安全”或“隐私”中，找到“安装未知应用”选项。
    *   允许当前文件管理器或浏览器安装应用。
4.  **执行安装**：点击 APK 文件，在弹出的提示框中点击“安装”。
5.  **权限授予**：安装完成后首次打开应用，请根据提示授予**相机权限**（用于拍照）和**存储权限**（用于读取相册图片）。

## 5. app启动

1.  **启动图标**：在手机桌面上找到带有绿色健康风格图标的“NutriScan”应用。
2.  **启动页**：点击图标进入应用，首先显示应用 Logo 和 Slogan。
3.  **登录/注册**：
    *   若用户未登录，将自动跳转至登录页面。
    *   支持手机号 + 验证码方式登录/注册。
    *   首次注册需完善个人资料（昵称、性别、生日、身高、体重、所属群体等）。
4.  **进入主页**：登录成功后，直接进入 APP 首页（识别页或社区页）。

## 6. app用法

APP 采用底部导航栏设计，分为四大主要模块：

1.  **识别（Scan）**：核心功能入口，支持拍照或从相册选择食物图片进行 AI 分析。
2.  **社区（Community）**：用户交流广场，浏览和发布健康饮食动态。
3.  **分析（Analysis）**：个人健康数据中心，展示 BMI、代谢数据及体重变化曲线。
4.  **我的（Profile）**：个人中心，管理个人信息、设置、查看历史记录等。

**通用操作逻辑**：
*   **点击**：选择菜单、查看详情。
*   **滑动**：浏览列表、切换图片。
*   **下拉刷新**：在社区或列表页更新内容。
*   **返回**：点击左上角返回图标或使用手机物理返回键。

## 7. 论坛功能

论坛（社区）模块是用户互动的核心区域，支持以下功能：

### 7.1 内容浏览与筛选
*   **分类标签**：顶部提供“养生”、“健身”、“幼儿”三个频道，用户可根据兴趣切换。
*   **搜索功能**：顶部搜索栏支持输入关键词（如“低脂”、“早餐”）搜索相关帖子。
*   **排序与过滤**：
    *   **已关注**：仅查看关注用户的动态。
    *   **最新/最热**：按发布时间或热度（点赞评论数）排序帖子。
*   **瀑布流展示**：帖子以双列瀑布流形式展示，封面图清晰直观。

### 7.2 帖子详情与互动
*   **查看详情**：点击帖子卡片进入详情页，支持查看多张轮播图片、完整文字内容及标签。
*   **点赞与收藏**：支持对帖子进行点赞（红心）和收藏（五角星），收藏内容可在个人中心查看。
*   **评论互动**：
    *   支持发表一级评论。
    *   支持对已有评论进行回复（二级评论）。
    *   支持展开/收起长评论区。
*   **关注作者**：在详情页可一键关注作者，获取其后续动态。

### 7.3 发布笔记
*   **发布入口**：点击右下角悬浮的绿色“+”号按钮。
*   **内容编辑**：
    *   **上传图片**：支持从相册多选图片（最多9张）。
    *   **标题与正文**：输入吸引人的标题和详细的饮食心得。
    *   **添加标签**：支持选择系统推荐标签（如#减肥、#高蛋白）或输入自定义标签。
*   **发布/草稿**：编辑完成后点击“发布”，或在退出时保存为草稿。

## 8. 食品识别功能

该功能利用大模型对食物进行深度解析，帮助用户把控饮食安全与营养。

### 8.1 识别操作
1.  **进入识别页**：点击底部导航栏的“相机”图标。
2.  **获取图片**：
    *   **拍照识别**：调用手机相机直接拍摄食物。
    *   **从相册选择**：从手机图库中选择已有的食物照片。
3.  **等待分析**：图片上传后，系统将进行 AI 处理，期间显示加载动画。

### 8.2 分析结果解读
识别完成后，系统会生成一份详细的分析报告，包含以下卡片：
*   **总览卡片**：显示识别到的场景、食物数量、过敏原数量及高风险项。
*   **识别结果详情**：
    *   **菜品名称**：AI 识别出的具体菜名。
    *   **主要原材料**：分析菜品的主要构成食材。
    *   **潜在过敏原**：**红色高亮**提示可能存在的过敏原（如花生、海鲜等）。
    *   **食用建议**：基于营养学的健康食用指导。
    *   **风险评估**：提示高糖、高油或其他健康风险。
*   **食物详情卡片**：针对盘中每一种食物的独立分析，包括置信度、预估份量、配料表及饮食标签（如“低碳水”、“高蛋白”）。
*   **全局冲突预警**：若食物包含常见过敏原或违反特定宗教/人群饮食限制，将醒目提示。

### 8.3 历史记录
*   用户可在识别页点击“查看识别历史”按钮，回顾之前的饮食分析记录。

## 9. 数据库软件

本项目后端采用混合数据库架构，以满足不同数据的存储需求：

1.  **MongoDB (NoSQL)**：
    *   **版本**：6.0+
    *   **用途**：主要用于存储非结构化和高并发的社交数据。
    *   **集合**：
        *   `posts`: 存储社区帖子内容、图片链接、点赞数等。
        *   `comments`: 存储用户的评论及回复层级结构。
        *   `users` (部分): 存储用户在社区维度的扩展信息。
    *   **优势**：Schema-free 特性适合存储结构多变的帖子和评论数据，读写性能高。

2.  **MySQL (关系型数据库)**：
    *   **版本**：8.0+
    *   **用途**：用于存储结构化强、事务要求高的核心用户数据。
    *   **表结构**：
        *   `app_user`: 用户基础信息（手机号、密码、身体数据等）。
        *   `sms_verification_code`: 短信验证码记录。
        *   `user_weight_log`: 用户体重历史记录。
        *   `user_calorie_log`: 用户热量摄入记录。

## 10. 数据库脚本文件

项目包含以下数据库初始化脚本，位于 `database/` 目录下：

### 10.1 `init_mongodb.js`
*   **类型**：MongoDB JavaScript 脚本
*   **功能**：
    *   创建 `nutriscan` 数据库。
    *   初始化 `users`, `posts`, `comments` 集合。
    *   **创建索引**：为关键字段创建索引以优化查询性能，例如：
        *   `posts` 集合上的 `category` (按分类查询)、`createdAt` (按时间排序)。
        *   `comments` 集合上的 `postId` (查询某帖子的评论)。
        *   `users` 集合上的 `phoneNumber` (唯一索引)。

### 10.2 `user_schema.sql`
*   **类型**：MySQL SQL 脚本
*   **功能**：
    *   创建核心关系型数据表。
    *   **`app_user` 表**：定义了用户ID、昵称、手机号、身高、体重、BMI、基础代谢等字段，并设置了主键和唯一约束。
    *   **`user_weight_log` 表**：用于存储用户体重随时间变化的记录，支持生成体重曲线图。
    *   **`sms_verification_code` 表**：用于管理验证码的发送、使用状态和过期时间。

## 11. 规范文件

为保证项目代码质量和团队协作效率，项目制定了严格的规范文件：

1.  **代码风格规范 (`.idea/codeStyles/Project.xml`)**
    *   统一了 Kotlin 和 XML 的代码格式化规则。
    *   规定了缩进（4空格）、换行、大括号位置等风格。
    *   开发人员在提交代码前需执行 `Reformat Code` 确保风格一致。

2.  **版本依赖规范 (`gradle/libs.versions.toml`)**
    *   采用 Gradle 的 **Version Catalog** 机制管理依赖。
    *   所有第三方库（如 Retrofit, Coil, Compose）的版本号统一在 `[versions]` 块中定义。
    *   库的坐标在 `[libraries]` 块中定义。
    *   **作用**：避免不同模块间依赖版本冲突，方便一键升级所有依赖库版本。

3.  **Git 忽略规范 (`.gitignore`)**
    *   配置了无需纳入版本控制的文件类型。
    *   包括：构建产物 (`/build`), 本地配置 (`local.properties`), IDE 缓存 (`.idea`, `.gradle`), 临时文件 (`.DS_Store`) 等。
